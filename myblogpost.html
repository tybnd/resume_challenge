My Journey into Cloud Engineering

When I first got into cloud engineering, I didn’t really know where to start. AWS, Lambda, APIs... it all seemed overwhelming. But like with anything, I realized that you just have to dive in headfirst, and learn as you go. Here are a few key moments that shaped my journey so far.

Lessons Learned

  • Start Small: My first project was a simple static website hosted on S3. This gave me a hands-on feel for what AWS can do without overcomplicating things.
  • Break Things (and Fix Them): Troubleshooting was—and still is—a huge part of my learning. I’ve spent hours debugging configurations and finding out why my Lambda function didn’t trigger. It’s frustrating but rewarding.
  • Master One Tool at a Time: I made the mistake of trying to learn everything at once. Focusing on one service (like Lambda or API Gateway) really helped me get comfortable before moving on to the next challenge.
  • Use Resources: AWS has some great docs, and the community is super helpful. Don't be afraid to ask questions and dig into resources like GitHub or StackOverflow.

What’s Next?

So, what’s next for me? I’m currently diving into Infrastructure as Code (IaC) using Terraform and AWS CloudFormation. The goal is to automate deployments and manage cloud infrastructure more efficiently. Stay tuned for more updates as I continue this cloud journey!
